# Skill: weekly-review

## ขั้นตอน

1. **รวบรวมข้อมูล** (Claudy ทำเองได้ — เป็นงาน orchestrator ไม่ใช่ specialist):
   - `worklog.json` → entries 7 วันล่าสุด จัดกลุ่มตาม agent + pipeline
   - `BACKLOG.md` → งานไหนขยับ/ค้าง/เลยกำหนด
   - สุขภาพระบบ: `git status` สะอาดไหม, dashboard อัปเดตล่าสุดเมื่อไหร่, news routine พลาดวันไหนไหม
2. **วิเคราะห์เทียบ success metrics** (CLAUDE.md):
   - Template ใหม่พร้อมขายเดือนนี้แล้วหรือยัง?
   - มีงานตกค้างที่ gate ไหนบ่อยสุด? (สัญญาณว่าต้องปรับ SOP หรือ scaffold)
3. **เขียนรายงาน** → `Output/Claudy/YYYY-MM-DD-weekly-review.md`:
   - เสร็จแล้ว (ชิ้น + agent) / ค้างอยู่ (ที่ gate ไหน เพราะอะไร) / ระบบ (hook, routine, deploy)
   - แผนสัปดาห์หน้า: 3-5 งานเรียงความสำคัญ พร้อม pipeline ที่จะใช้
4. **อัปเดต `BACKLOG.md`**: ติ๊กงานเสร็จ, เลื่อนลำดับ, เพิ่มงานใหม่ที่โผล่ระหว่างสัปดาห์
5. Commit + push + รายงานสรุปสั้นให้ Kittanate (5-10 บรรทัด ลิงก์ไฟล์เต็ม)

## หลักการ
- รายงานต้อง actionable: ทุกปัญหาที่ยกมา ต้องมีข้อเสนอถัดไป 1 ข้อเสมอ
- อย่าปั้นตัวเลขให้ดูดี — สัปดาห์ที่ไม่มีอะไรเสร็จก็รายงานตามจริงพร้อมเหตุผล
