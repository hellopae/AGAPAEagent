# SOP-10 — Visual Design Standards (ใช้ Claude Design / `artifact-design`)

> เจ้าของ: **Mind** (visual) + **Vera** (layout) | Gate: **Chris** (SOP-08 D)
> เกิดจากบทเรียนงาน VeeGee Box (7 ก.ค. 2569): ออกแบบผิดขนาด + ไม่ใช้ภาพจริงที่มีอยู่ + ได้แค่ mockup ไม่ print-ready
> หลักการ: **ต้นทางจริงมาก่อนการเดา — คุณภาพต้องเทียบชั้นภาพอ้างอิงเสมอ**

## 0. STEP 0 — สำรวจ asset ต้นทางก่อนออกแบบ (บังคับ ห้ามข้าม)

ก่อนวาด/จัดเลย์เอาต์ใด ๆ ให้ `Read`/`ls` โฟลเดอร์โปรเจกต์ต้นทางทั้งหมดก่อน แล้วทำ **Asset Inventory**:

- [ ] **Dieline / แม่แบบขนาด** — มีไฟล์กล่อง/เทมเพลตจริงไหม (เช่น `Box4.jpg/.pdf`) → **ล็อกขนาดจากไฟล์นี้เท่านั้น** ห้ามสมมติเอง
- [ ] **ภาพถ่ายสินค้าจริง** — มีในโฟลเดอร์ (`Product/`, `photos/`) ไหม → **ถ้ามี ต้องใช้ภาพจริง ห้ามวาด vector แทน**
- [ ] **โลโก้ vector** — `.ai/.eps/.svg/.png` ตัวจริง → ใช้ไฟล์จริง ไม่วาดเลียนแบบ
- [ ] **ภาพอ้างอิง/mood (`Exam/`, references)** — คือ "ระดับที่ต้องไปให้ถึง" → ตั้งเป็น quality bar
- [ ] ระบุ **สิ่งที่ยังขาด** ให้ชัด แล้วรายงาน Claudy — อย่าเดาแทนของที่ไม่มี

> ⛔ กับดักที่เคยพลาด: บันทึกว่า "ไม่มีภาพในโฟลเดอร์" ทั้งที่มี / อ้างขนาด die-cut ที่ไม่ตรงกล่องจริง — Step 0 มีไว้กันเรื่องนี้

## 1. ใช้ "Claude Design" เป็นเครื่องมือหลัก

ทุกงาน visual ให้เรียก skill **`artifact-design`** ก่อนลงมือ เพื่อ calibrate ระดับดีไซน์ (สี, ลำดับชั้น, ระยะ, การอ่านออกทั้ง light/dark) แล้วผลิตเป็น artifact HTML self-contained

- ฟรี ไม่มีลิมิตบัญชีภายนอก — เป็น default ของทีม
- Embed asset จริงเป็น `data:` URI (โลโก้/ภาพถ่าย) ให้เปิดออฟไลน์ได้
- คุณภาพต้อง **เทียบชั้นภาพอ้างอิง** ไม่ใช่ "พอเป็น mockup"

**เมื่อไรถึง escalate ออกจาก Claude Design:** ถ้างานต้อง **vector print จริง / dieline แม่นยำ / ไฟล์ CMYK ส่งโรงพิมพ์** → รายงาน Claudy ให้พิจารณา Figma MCP (vector/dieline) หรือ Canva MCP (ต่อบัญชีผู้ใช้) เป็น step ผลิตไฟล์พิมพ์จริง โดย proof คุมทิศทางยังทำใน Claude Design ได้

## 2. Print artwork — ล็อกให้ตรงของจริง

- ขนาด/สัดส่วน panel = **จากไฟล์ dieline ต้นทาง** (เช่น VeeGee = กว้าง 11 × ยาว 24 × สูง 10 cm)
- bleed 3mm ทุกด้าน, safe margin ≥5mm, ทุกสีเป็น CMYK, 300dpi
- โลโก้/ภาพ = ไฟล์จริงความละเอียดพอ ไม่ยืดเบลอ

## 3. Self-check ก่อนส่ง Chris (Mind/Vera ทำเองก่อน)

- [ ] ขนาดตรง dieline ต้นทางจริง (เทียบไฟล์ ไม่ใช่ตัวเลขในหัว)
- [ ] ใช้ asset จริงครบ (ภาพถ่าย + โลโก้ vector) — ไม่มีของวาดแทนทั้งที่มีของจริง
- [ ] คุณภาพเทียบชั้นภาพอ้างอิงแล้ว
- [ ] ระบุ asset ที่ยังขาด + ใครต้องส่งมา (เช่น regulatory จากลูกค้า)

## 4. Deliverable

`Output/Mind/YYYY-MM-DD-<slug>-artwork.html` (artifact) + `.md` proof note ที่มี: Asset Inventory, ขนาดที่ล็อก, asset จริงที่ใช้, สิ่งที่ยังขาด, ลิงก์ artifact
