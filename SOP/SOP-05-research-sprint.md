# SOP-05 — Research Sprint: ตอบคำถามตลาด/ข้อมูลแบบมีหลักฐาน

> ใช้เมื่อโจทย์คือคำถาม เช่น "printable แบบไหนขายดีในไทย", "ควรตั้งราคาเท่าไหร่",
> "เทรนด์งานพิมพ์ปีนี้เป็นยังไง"

## Pipeline

```
Reese [Research] ──┐
                   ├→ Reese [Fact-check รวม] → Chris QA → Libby (จัดเก็บรายงาน) → ส่งมอบ
Nick (ข้อมูลภายใน) ─┘
```

Reese กับ Nick ทำงาน **ขนานกัน** ได้ — Reese มองออกนอก (ตลาด/คู่แข่ง), Nick มองเข้าใน (ยอดขาย/ข้อมูลเรา)

## ขั้นตอนละเอียด

### ขั้น 1 — Claudy แตกคำถาม
แปลงคำถามใหญ่เป็น sub-questions ที่ตอบได้จริง เช่น
"printable แบบไหนขายดีในไทย" →
- หมวดไหนมี listing/review มากสุดบน Etsy เมื่อค้นด้วย keyword ไทย
- ช่วงราคาที่ขายได้จริง
- ฤดูกาล/เทศกาลไหนดันยอด
- ข้อมูลขายของเราเอง (ถ้ามี) บอกอะไร

### ขั้น 2 — Reese [Research]
- ทุกตัวเลข/ข้ออ้างต้องมี source แนบ — "ไม่พบข้อมูล" เป็นคำตอบที่ยอมรับได้ การเดาไม่ใช่
- แยกชั้นความมั่นใจ: ยืนยันแล้ว / แนวโน้ม / สมมุติฐาน
- Output: `Output/Reese/YYYY-MM-DD-<slug>-research.md`

### ขั้น 3 — Nick (ถ้ามีข้อมูลภายใน)
- วิเคราะห์เฉพาะข้อมูลที่มีจริง (CSV, ยอดขาย, analytics) — sample เล็กเกินไปให้บอกตรง ๆ
- Output: `Output/Nick/YYYY-MM-DD-<slug>-analysis.md`

### ขั้น 4 — Reese [Fact-check] รายงานรวม
- Fact-check ทั้งรายงานของตัวเอง (mode แยก) และของ Nick — ตัวเลขทุกตัวต้องมีที่มา
- Output: `Output/Reese/YYYY-MM-DD-<slug>-factcheck.md`

### ขั้น 5 — Chris QA + ขั้น 6 — Libby
- Chris: ตรรกะรายงาน, ข้อสรุป overclaim ไหม, actionable ไหม
- Libby: จัดเก็บเข้า index ให้ค้นเจอภายหลัง (research reports คือ asset ระยะยาว)

## มาตรฐานรายงาน Research

โครงบังคับ:
1. **คำตอบสั้น** (3-5 บรรทัด ตอบคำถามตรง ๆ ก่อน)
2. หลักฐานรายข้อ พร้อม source
3. สิ่งที่ยังไม่รู้ / ข้อจำกัดของข้อมูล
4. **Action ที่แนะนำ** — ส่งต่อให้ Minnie ต่อยอดเป็นไอเดียได้ทันที (knowledge loop)
